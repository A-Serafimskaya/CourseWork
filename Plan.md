## 1. Перечень автоматизируемых сценариев 

**Под валидными данными будут пониматься следующие значения:**

* Для поля "Номер карты": числовое значение,состоящее из 16-ти символов
* Для полей срока карты ("Месяц", "Год"): числовое значение будущей даты, которая еще не наступила, формата ХХ каждая
* Для поля "Владелец": буквенное выражение на латинице вида "Имя Фамилия" или "И. Фамилия". Не считая пробела, может включать в себя ТОЛЬКО символы "-" и "."
* Для поля "CVC/CVV": числовое значение, состоящее из 3-х символов

## 2. Используемые инструменты для реализации сценариев

| Инструмент        | Обоснование выбора                                                                                               |
|-------------------|------------------------------------------------------------------------------------------------------------------|
| 1. Selenide       | Относительно простой в использовании фреймворк для автоматизации тестирования                                    |
| 2. JUnit 5 (java) | Фреймворк для юнит-тестирования, облегчающий автоматизацию тестирования наличием широких возможностей            |
| 3.Gradle	     | Современная система сборки и автоматизации проектов, отличается гибкостью и высокой производительностью          | 
| 4. Allure Report  | Простой и понятный генератор отчетов о тестировании, есть минимальный необходимый набор функций                  |
| 5. Lombok         | Библиотека, генерирующая код, упрощает работу разработчику                                                       |
| 6. Docker         | Упрощает разработку и тестирования в виду возможности использования маловесных изолированных сред для приложений |
| 7. Faker          | Генератор тестовых данных                                                                                        | 


## 3. Возможные риски при автоматизации
1. Изменения в структуре и дизайне веб сервиса: может привести к нарушению работы селекторов в тестах
2. Нестабильная работа зависимых систем при тестировании
3. Нечеткось в требованиях и документации
3. Ошибки в самих тестах
